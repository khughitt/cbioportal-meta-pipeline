# WeSME: uncovering mutual exclusivity of cancer drivers and beyond

- **Authors:** Yoo-Ah Kim, Sanna Madan, Teresa M. Przytycka
- **Year:** 2017 (Bioinformatics issue date; originally published online 2016)
- **Journal:** Bioinformatics 33(6):814-821
- **DOI/URL:** https://doi.org/10.1093/bioinformatics/btw242
- **PMID:** 27153670
- **OpenAlex ID:** W2963382551
- **BibTeX key:** Kim2017
- **Source:** OpenAlex + PubMed (verified 2026-04-13)
- **Method name:** **WeSME** (Weighted Sampling Mutual Exclusivity)

## Key Contribution

Introduces **WeSME**, a weighted-sampling permutation framework for mutual-exclusivity
testing that **preserves per-sample mutation rates** while shuffling alteration assignments.
Scales well to large pan-cancer cohorts and does not require a closed-form null. Provides a
companion scheme (WeSCO) for co-occurrence. Empirically shown to control false-positive
rates comparably to DISCOVER while being more robust to extreme hypermutators.

## Methods

- **Null model:** sampling scheme in which, for each simulated matrix, each gene's
  alteration pattern is redrawn conditional on per-sample alteration probabilities derived
  from the observed marginals.
- **Acceleration:** pre-computed tables of sampling weights per sample to avoid full matrix
  permutation.
- **Pair test + set test:** extends to groups of genes without combinatorial explosion.

## Key Findings

- Recovers DISCOVER's main exclusivity calls on TCGA pan-cancer data with higher power for
  sparsely altered genes.
- Better behavior on rare-alteration pairs than Fisher-exact or naive permutation.
- Companion co-occurrence test (WeSCO) finds very few co-occurrences survive after
  sample-burden correction, consistent with Canisius 2016.

## Relevance

The pragmatic alternative to DISCOVER when closed-form Poisson-binomial math is awkward or
when the project wants more explicit control over the null via per-sample weights. For the
cbioportal pipeline, WeSME is attractive because (a) it integrates cleanly with the
existing Snakemake rule structure (a permutation-sampling step is trivially parallelizable
per study), and (b) it avoids relying on the Poisson-binomial approximation for small /
sparse per-study matrices.

## Limitations

- Permutation-based — slower than DISCOVER for very large cohorts; moot at the project's
  current scale (tens of thousands of samples, not millions).
- No built-in covariate adjustment for lineage (unlike SELECT).
- Not designed for cross-study meta-analysis — the project would need to combine
  per-study WeSME *p*-values explicitly (Stouffer / Fisher / random-effects).
