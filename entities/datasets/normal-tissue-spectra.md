## Aggregation definitions

- **pooled_counts** — column-wise sum of per-donor 96-context counts across all donors for the tissue. Integer counts. Highest-burden donor dominates. One row per tissue.
- **donor_averaged_fraction** — per-donor 96-vector normalised to a fraction (sum = 1), then averaged across donors, re-normalised to guard against rounding drift. Fractions summing to exactly 1. Donors with fewer than `low_snv_threshold` (default 50) SNVs in the tissue are excluded; excluded counts surface on the output row via `n_donors_excluded_low_snvs` / `low_snv_threshold`. One row per tissue.
- **per_donor** — raw per-donor 96-context counts. Integer counts. One row per donor per tissue.

## SBS-only filtering

The SigProfiler SBS96 matrix is the only output channel. Indels and DBS variants are filtered out. The 2026-04-19 extraction run dropped 583 rows flagged as indels by `validate_input_contract` (length-1 check, plus `-` sentinel used by the Li 2021 supplement for insertions/deletions). DBS variants (reported by SigProfiler as e.g. "21 DINUCs") are enumerated in the pre-matrix summary but not retained in our output — only SBS96 is extracted.

## Replication

```bash
# 1. Stage the Li 2021 XLSX (one-off):
curl -sSL -o data/41586_2021_3836_MOESM5_ESM.xlsx \
    "https://static-content.springer.com/esm/art%3A10.1038%2Fs41586-021-03836-1/MediaObjects/41586_2021_3836_MOESM5_ESM.xlsx"
sha256sum data/41586_2021_3836_MOESM5_ESM.xlsx  # expect 6276deb0...

uv run python code/scripts/stage_li2021_somatic_mutations.py \
    --xlsx data/41586_2021_3836_MOESM5_ESM.xlsx \
    --out data/li2021_somatic_mutations.tsv

# 2. Run the extraction:
uv run snakemake -s code/workflows/Snakefile --configfile code/config/config-10k-genes.yml -j1 \
    data/normal_tissue_spectra.tsv data/normal_tissue_burden.tsv
```

Requires:
- Internet access on first run (SigProfilerMatrixGenerator downloads GRCh37 reference bundle ~780 MB to the user cache; uncompresses to ~3.1 GB)
- `data/tissue_uberon_mapping.tsv` (committed to git)
